# Documentation

## Purpose

Additional documentation including:
- Architecture diagrams
- Development guides
- API documentation
- Troubleshooting guides
- Design decisions

## Structure

```
docs/
├── architecture/       # Architecture diagrams and explanations
├── guides/             # Development and user guides
├── api/                # API documentation
├── troubleshooting/    # Common issues and solutions
└── decisions/          # Architecture decision records (ADRs)
```

## Content Guidelines

- Use Markdown for all documentation
- Include diagrams using Mermaid
- Keep guides practical and example-driven
- Update docs alongside code changes
- Use ADRs for significant architectural decisions
